01

Game Pitch

Short AR sessions where a mascot guides 2–3 mini‑games. The child’s motions drive a mirroring avatar; tokens and calm praise reward effort; light telemetry adapts difficulty.

02

Core Loop & Session Flow

  1. Calibrate & scan → draw bounds.
  2. Mascot intro → “today’s quest”.
  3. Rounds (60–120s) with tokens & avatar response.
  4. Calm break (3–5 breaths) as needed.
  5. Summary: tokens, highlights, optional CSV export.
03

Avatar Mirroring Spec

Input Pipeline
  • Primary: touch + device orientation for big targets.
  • Optional: pose proxy (wrist/hip) when available.
Mapping
  • Reach → arm IK to target; Place → basket drop.
  • Step L/R → avatar steps; mirrored animation.
  • Latency budget < 150 ms.
04

Mechanics (Design Sheets)

Color Bubbles — Reach & Tap
  • Goal: reach height & accuracy.
  • Setup: shoulder→overhead bubbles; generous colliders.
  • Success: tap/reach; pop + token.
  • Knobs: size, height, distance, time.
  • Telemetry: hits/misses, max height, reaction/move ms.
Trail Steps — Balance & Gait
  • Goal: bilateral stepping/balance.
  • Setup: floor pads L/R; sequence.
  • Success: pad contact/step proxy.
  • Knobs: spacing, length, tempo.
  • Telemetry: L/R counts, cadence, misses.
Catch & Place — Grasp & Precision
  • Goal: hand–eye coordination.
  • Setup: floating objects; color‑match baskets.
  • Success: pick → carry → place.
  • Knobs: speed, basket size/distance, time.
  • Telemetry: accuracy, time to place, path proxy.
Mirror Moves — Bilateral Mapping
  • Goal: pose imitation.
  • Setup: mascot shows 2–4 s poses.
  • Success: hold within tolerance.
  • Knobs: hold time, poses count, L/R mix.
  • Telemetry: holds, drift, side balance.
Calm Cave — Regulation
  • Goal: down‑shift arousal.
  • Setup: breathing fireflies; 3–5 cycles.
  • Success: cycles completed.
  • Knobs: tempo, cycles, intensity.
  • Telemetry: break count/duration, pre/post ease rating.
05

Difficulty & Adaptation

  • ≥80% success ×2 → harder (+10% distance/height or −10% size).
  • <60% → easier (reverse); add hint/VO.
  • Reduce timers 5–10% only after 3 consecutive wins.
  • Manual override sliders; all changes logged.
06

UX, UI & Sensory Spec

UI
  • Large buttons; progress dots; “next up”.
  • Token board; sticker chooser.
Visual
  • Saturated‑soft palette; high‑contrast toggle.
  • Predictable, slow animations.
Audio
  • Warm VO; soft success tones; no harsh fails.
  • Global volume + Quiet Mode.
07

Telemetry Schema

Per round
  • game_id, round_id, mechanic_id, start_ts, end_ts
  • targets_shown, hits, misses, max_height, distance
  • reaction_ms, movement_ms
  • side_L, side_R, midline_crosses
Per session
  • session_id, duration_sec, breaks_count, breaks_sec
  • difficulty_params (size, distance, time)
  • exported_csv (bool), consent_version
Local‑first; optional anonymized CSV. No PII without consent.
08

Safety & Caregiver Controls

  • AR bounds; out‑of‑bounds nudge.
  • Pause/skip/end; difficulty sliders; session timer.
  • Quiet mode; high‑contrast; reset anchors.
09

Build Plan & Roadmap

MVP (6–8 wks)
  • Plane detection; anchors; URP.
  • Mascot + avatar mirroring (pose proxy).
  • Color Bubbles, Trail Steps, Catch & Place, Calm Cave.
  • Tokens; logs; caregiver controls.
V1.0
  • Mirror Moves; profiles; adaptation service.
  • Dashboard export; sensory presets.
  • Home “Lite” mode.
Risks
  • Tracking drift → big colliders; re‑anchors.
  • Overstimulation → quiet defaults; caps.
  • Licensing → original characters.
10

QA & Acceptance Criteria

  • Playable in 2×2 m space; clear bound warnings.
  • Avatar latency <150 ms on target devices.
  • 10–15 min sessions with no hard blocks.
  • CSV export validates against schema.
11

Non‑Goals

  • Clinical diagnosis without approvals.
  • Competitive leaderboards.
  • Complex multi-user AR.
12

Glossary

  • Avatar mirroring: mapping child motions to on‑screen character.
  • DCD: Developmental Coordination Disorder.
  • Quiet Mode: reduced audio/visual intensity preset.
